# Alpha Hunter 因子猎手 —— 自研遗传编程因子挖掘引擎（v0.1.0）

> “我们不制造因子，我们只是市场的狩猎者。”

---

## 项目定位

**Alpha Hunter** 是一个从零手写的遗传编程（Genetic Programming）因子挖掘引擎。它不是 `gplearn` 的封装，而是一个专为金融量化因子挖掘设计的**轻量级、高可控性**框架。

**当前状态**：v0.1.0 —— 核心挖掘流水线已跑通，已能产出稳定有效的因子。

**核心理念**：让计算机在纯净的数学空间中自由组合特征，通过"自然选择"筛选出真正有效的 Alpha 因子，拒绝常数硬编码、拒绝黑箱过拟合。

##  特征说明

引擎预设了 4 个基础特征，用户可自行扩展：

| 特征 | 符号 | 计算方式 | 含义 |
|------|------|----------|------|
| 昨日收益率 | X0 | close / close.shift(1) - 1 | 短期动量 |
| 振幅 | X1 | (high - low) / close | 日内波动 |
| 量比 | X2 | volume / MA(volume, 5) | 相对成交量 |
| 日内强度 | X3 | close / open - 1 | 日内多空力量 |

##  运算符池

| 类型 | 运算符 | 说明 |
|------|--------|------|
| 二元 | add, sub, mul, div | 四则运算 |
| 一元 | neg, inv, log, sqrt, square | 非线性变换 |

> ⚠️ **注意**：所有一元运算符均内置数值保护（如 log 取绝对值，div 防除零），确保评估过程不产生 NaN 或 inf。

##  内置过滤器（免疫系统）

引擎在生成阶段会自动拦截以下垃圾结构：

| 拦截类型 | 示例 | 原因 |
|----------|------|------|
| 定义域错误 | log(neg(X)), sqrt(neg(X)) | 数学未定义 |
| 代数抵消 | add(neg(X), X), sub(X, X) | 恒为常数 |
| 双重否定 | neg(neg(X)), inv(inv(X)) | 等价于原特征 |
| 逆运算抵消 | div(mul(X,Y), Y) | 等价于 X |
| 连续压缩 | log(log(log(X))) | 方差趋近于零 |
